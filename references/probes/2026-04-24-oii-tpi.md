## Comparison / handler recommendation

`tpi.li` appears to be the same token-embedded Turnstile/timer family as `oii.la`, not a fundamentally new flow:

- Both are `200 OK` entry pages with `ref<alias>` cookie.
- Both expose a single POST form to `advertisingcamps.com/taboola{N}/landing/`.
- Both use Turnstile, `counter_value=15`, `counter_start=load`, `captcha_shortlink=yes`.
- Both hide the real faucet `links/back` URL in a base64 tail inside hidden `token`.
- Both advertisingcamps POST checks returned `302 -> https://www.taboola.com`, so the landing endpoint should not be treated as the oracle.

Next handler recommendation:
1. Add `tpi.li` to the same analysis-only handler lane as `oii.la` (or rename/generalize it to a token-tail Turnstile family handler).
2. Reuse `_extract_runtime_config`, `_extract_hidden_inputs`, `_collect_embedded_urls`, and `_extract_oii_token_target` logic unchanged for `tpi.li`; the extractor already fits the token shape.
3. Keep result notes honest: return embedded oracle as `EMBEDDED_TARGET_EXTRACTED`, but do not claim live captcha/timer completion.
4. Optional facts to expose for this family: `form_action`, `hidden_inputs`, `token_target`, `captcha_type`, `sitekey`, `counter_value`, `counter_start`, and `ad_handoff_redirect=https://www.taboola.com` if a POST check is later implemented.
