## Timer boundary

Observed final `/links/go` responses in the same session:

```text
0s after final page load  -> Bad Request
5s after final page load  -> Bad Request
~17s after final page load -> success
```

The page displays a 15-second timer. Safe handler wait recommendation: `16s` or slightly more from successful final `go-link` page fetch.

## Handler recommendation

Add an `lnbz.la` browserless handler using plain `requests` or `curl_cffi` if TLS issues appear later. The robust path should:

1. start a fresh session with browser-like headers
2. `GET https://lnbz.la/<alias>`
3. parse and submit the entry form to the provided `avnsgames.com` action
4. parse and submit each `form#go_d2` until the action returns to `https://lnbz.la/<alias>`
5. parse final `form#go-link`
6. wait at least `16s` from final page load
7. `POST /links/go` with same-session hidden fields and AJAX-style headers
8. only return success when JSON `status=success` and `url` is present

Do not rely only on the decoded token as final proof. Token decoding is a useful fallback/explanation, but the success oracle is `/links/go` returning the exact downstream URL.
