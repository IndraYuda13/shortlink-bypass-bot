# 2026-04-28 XUT hybrid integration attempt

## Goal
Try to reduce `xut.io/hd7AOJ` runtime by moving the browser-independent parts to HTTP while keeping browser only for `gamescrate.app`.

## Attempted implementation
Built an experimental local helper `xut_http_hybrid.py` with:
- HTTP xut entry -> autodime Step 1
- HTTP IconCaptcha LOAD/SELECTION/verify
- HTTP autodime/textfrog Step 2-4
- browser only for gamescrate Step 5
- HTTP xut Step 6 `/links/go`

## Live result 1
- HTTP warmup reached `gamescrate.app/cwsafelinkphp/setcookie.php?t=...`.
- Browser launched directly on that gamescrate setcookie URL.
- Result: gamescrate body was `Forbidden.` and no Step 5 page appeared.
- Total failed time: `286.9s`.

## Live result 2
- Updated HTTP UA to match installed Chrome major (`147`) instead of static Chrome 136.
- HTTP warmup still reached gamescrate setcookie.
- Browser still received `Forbidden.` on gamescrate.
- Total failed time: `282.8s`.

## Interpretation
The signed gamescrate token generated by HTTP Step 2-4 appears bound to the HTTP/client fingerprint. Opening it later in a different Chrome/browser context is rejected by gamescrate. This falsifies the simplest `HTTP Steps 1-4 -> browser Step 5` hybrid.

## Extra probe
Tried a second shape: HTTP only through IconCaptcha/Step 1, then browser from Step 2 so later tokens would be generated by browser. The run exceeded the timeout and was killed before a final oracle, so this shape is not proven.

## Current decision
Do not wire `xut_http_hybrid.py` into production yet. The existing full-browser xut lane remains the only proven success lane.

## Next possible experiments
1. Browser from Step 2 with longer timeout and better progress logging.
2. Browser-started session with HTTP IconCaptcha solved by exporting/importing cookies into the same browser context.
3. Investigate what fingerprint claim inside the signed `t=` token is compared by gamescrate.
